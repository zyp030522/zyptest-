git 分支操作

查看分支列表
git branch
分支前面的*表示当前所处的分支

创建新的分支
git branch 分支名称

切换分支操作命令
git checkout 分支名称

分支快速创建和切换
git checkout -b 分支名称
-b 表示创建一个分支
checkout 表示切换到刚才创建的分支上

合并分支    注意 **合并分支时要先切换到主分支上才可以进行
git merge 要合并到主分支的分支名称

删除分支   注意如果删除的分支还没有合并到主分支上时 会报错 提示强行删除的方式 git branch -D 
git branch -d 分支名称

遇到冲突时的分支合并   需要手动解决冲突
冲突情况提示代码
CONFLICT (content): Merge conflict in index.js
Automatic merge failed; fix conflicts and then commit the result.
解决方法：打开包含冲突的文件 手动解决冲突后  再执行下面的操作
git add .
git comit -m "解决了分支合并冲突的问题"

将本地分支推送到远程仓库
git push -u origin 本地分支的名称:远程分支名称
实际案例
git push -u origin payment:pay
如果希望远程分支和本地分支的名称保持一致可以简化
git push -u origin payment
注意：第一次将本地分支推送到远程分支 需要带-u 参数 
          此后可以直接使用git push 推送代码到远程分支

查看远程仓库中所有的分支列表
git remote show 远程仓库名称

跟踪分支  从远程仓库中 分支下载到本地仓库中
从远程仓库中 把对应的远程分支下载到本地仓库中 保持本地分支和远程分支的名称一致
git checkout 远程分支的名称
示例  git checked pay
从远程仓库中 把对应的远程分支下载到本地仓库中 并把下载的本地分支进行重命名
git checkout -b 本地分支名称 远程仓库名称/远程分支名称
示例 git checkout -b payment origin/pay

拉去远程分支的最新代码
从远程仓库 拉去当前分支的最新代码 保持当前分支代码和远程分支代码一致
git pull

删除远程分支
删除远程仓库中指定名称的远程分支
git push 远程仓库名称 --delete 远程分支名称
示例   git push origin --delete pay
